# MusicKeyboard Project Direction

정리 날짜: 2026-04-16

## 배경

본 프로젝트는 남아 있는 `RP2350` 자원을 활용해, 커스텀 키보드 기반의 새로운 음악 인터페이스를 개발하는 데서 출발한다. 초기 아이디어는 컴퓨터나 외부 음원 장치를 제어하는 `MIDI 입력 인터페이스`였으며, 이후 자체 스피커와 샘플 재생 기능을 포함한 `독립형 로파이 샘플러 악기`로의 확장 가능성을 확인하였다. 이에 따라 본 프로젝트는 하나의 하드웨어 플랫폼을 단계적으로 확장하는 방식으로 접근한다.

추가로, `Ulanzi D200 + RP2350` 조합을 2단계 하드웨어 플랫폼으로 확정하였다. D200의 5인치 LCD(720×1280)와 13개 버튼을 UI/디스플레이로 활용하고, RP2350이 오디오 DSP와 D200 UART 통신을 모두 담당하는 구조다.

## 목표

`1단계`의 목표는 `RP2350 기반 MIDI Controller Interface MVP`를 구현하는 것이다. 이는 키 입력을 읽고 USB MIDI 또는 외부 MIDI 신호를 안정적으로 송신하는 입력 장치다.

`2단계`의 목표는 이를 `Standalone Lo-Fi Sampler Instrument`로 확장하는 것으로, 마이크 입력, 샘플 녹음/재생, 재생 속도 기반 피치 변화, 자체 오디오 출력을 포함하는 독립형 전자악기를 구현하는 데 있다.

## 단계별 범위

`1단계`에는 키 입력 스캔, 디바운싱, USB MIDI 송신, 필요 시 TRS/DIN MIDI 출력, LED 또는 간단한 UI 피드백을 포함한다.

`2단계`에는 마이크 입력, 짧은 샘플 녹음 및 저장, 재생 속도 변화 기반 피치 조절, 소형 스피커 또는 3.5mm 출력, 기본적인 trim/filter/bitcrush 기능을 포함한다. UI는 D200 5인치 화면 + 13버튼으로 구성한다.

반면 고음질 스테레오 엔진, 고급 time-stretch, 복잡한 DAW 수준 편집 기능은 범위에서 제외하여 구현 가능성을 높인다.

## 기술적 난점 및 리스크

`1단계`의 핵심 난점은 입력 스캔 안정화, MIDI 호환성, 지연시간 관리 등으로 비교적 예측 가능하다.

반면 `2단계`는 실시간 오디오 버퍼 처리, 저장공간 및 메모리 관리, 노이즈와 전원 안정성, 스피커 피드백, 기구 구조와 음향 성능의 상호 영향까지 함께 다뤄야 한다. 즉, `1단계`가 이벤트 기반 제어 장치 개발이라면, `2단계`는 실시간 오디오 시스템 통합 개발이라는 점에서 난이도가 크게 상승한다.

## 개발 전략

개발은 `1단계`를 최소기능제품(`MVP`)으로 우선 완성하고, 이를 바탕으로 `2단계`를 확장형 플랫폼으로 추진하는 전략을 채택한다. 이 방식은 초기 성공 기준을 명확히 하고 기술 리스크를 분산시키며, 동일한 하드웨어 기반 위에서 점진적으로 기능을 확장할 수 있다는 장점이 있다.

특히 `2단계`에서는 `속도 변화 = 피치 변화` 방식의 로파이 샘플러 구조를 우선 채택해, 구현 복잡도를 낮추면서도 독립형 악기로서의 정체성을 확보하는 것을 핵심 방향으로 설정한다.

## 2단계 하드웨어 구성 (확정)

```
D200 (RK3308 Linux)
  ↕ UART 1.5Mbps / NDJSON
RP2350
  ↕ I2S                  ↕ I2S
INMP441/ICS43434 마이크  MAX98357A + 8Ω 스피커
```

### D200 UART 프로토콜

양방향 모두 NDJSON, `\n` 구분자.

**RX (D200 → RP2350)**: 버튼 이벤트
```json
{"type":"button_press","id":0}
{"type":"button_release","id":0}
```

**TX (RP2350 → D200)**: 화면 업데이트
```json
{"type":"set_button","id":0,"text":"120BPM","icon":5}
{"type":"set_button","id":1,"value":0.75,"color":"#FF0000"}
```

프로토콜 레퍼런스: [mapero/esphome-ulanzi-d200 commands.go](https://github.com/mapero/esphome-ulanzi-d200/blob/main/daemon/pkg/transport/commands.go)

### D200 버튼 레이턴시

```
버튼 누름 → Linux input event → ulanzi-control 데몬 → UART → RP2350
                                                      합계: 2~5ms
```

로파이 샘플러 용도에서 충분히 허용 가능한 범위다.

### ESP32-S3-ETH는 왜 쓰지 않나

샘플러는 네트워크가 불필요한 스탠드얼론 기기다. ESP32를 끼우면 구조만 복잡해진다. ESP32-S3-ETH는 DMX, 센서 수집, 모니터링 등 네트워크가 필요한 별도 프로젝트([Esp32-d200](../Esp32-d200/))에 활용한다.

### GPIO 사용 계획

| 용도 | 핀 수 |
|------|-------|
| UART to D200 (TX/RX) | 2 |
| I2S 마이크 (SCK/WS/SD) | 3 |
| I2S 출력 MAX98357A (BCLK/LRC/DIN) | 3 |
| SD카드 샘플 저장 SPI (선택) | 4 |
| **합계** | **12** (18개 여유) |

## 1단계 브링업 현황 (2026-04-17 기준)

RP2350 bringup 펌웨어(`firmware/rp2350_bringup`)가 하드웨어에서 동작 확인됨.

### 확인된 기능

- USB 시리얼 디버그 출력
- SPI microSD + FAT32 WAV 로드 (`samples/` 하위 폴더 포함)
- PWM 오디오 출력 (GP15)
- 4개 블루 버튼 → C-Am-F-G 3보이스 코드 재생 (retrigger 포함)
- 빨간 버튼 → 루프 녹음 시작/정지 (트랜스포트 연동)
- 온보드 LED (GP25 PWM) → 녹음 상태 시각 피드백
